**프로젝트명:** Skills 기반 투자 데이터 대시보드  
**문서 버전:** 1.1 (구현·Skills 동기화)  
**작성 목적:** 해커톤 제출용 기획서 (PDF 변환 원본)

## 1. 서비스 개요

### 1.1 배경 및 문제 정의

금융·투자 업무에서는 CSV·엑셀 등으로 넘어오는 시계열 가격 데이터의 **컬럼명·형식이 제각각**이어서, 매번 동일한 분석을 하더라도 전처리와 화면 구성을 반복 구현해야 합니다. 분석 기준이 문서화되지 않으면 **팀 내 해석 불일치**와 **대시보드 품질 편차**가 발생합니다.

### 1.2 서비스 한 줄 정의

**`Skills.md`에 정의한 분석·시각화·인사이트 규칙을 소프트웨어가 그대로 따르는** 웹 기반 투자 데이터 대시보드입니다. 사용자는 CSV를 업로드하기만 하면, 규칙 문서에 따라 컬럼 매핑·지표 계산·차트 배치·경고 메시지가 **자동으로** 구성됩니다.

### 1.3 목표 사용자

- 내부용으로 빠르게 시계열 수익·리스크를 확인해야 하는 **애널리스트·기획·개발** 역할
- 해커톤·교육 맥락에서 **데이터 구조가 바뀌어도 동일 규격**으로 시연하고 싶은 참가자

### 1.4 제공 가치

| 가치 | 설명 |
|------|------|
| 범용성 | `date`·`close` 등 역할별 **별칭 목록** + **UTF-8/CP949 자동 판별** + **키움 8열 그리드 위치 폴백**으로 증권 HTS에서 보낸 CSV에 대응 |
| 일관성 | 지표 정의·차트 선택·인사이트 문구가 **단일 문서(Skill)**에 집중 |
| 자동화 | 업로드 한 번으로 KPI·복수 차트·인사이트까지 **연쇄 생성** |
| 투명성 | 해석 가드레일(비권유·과거 성과 한계)을 문서와 UI에 명시 |

### 1.5 범위 및 전제

- **1차 범위:** 단일 시계열 **종가(또는 관측 시점 가격) 중심** 데이터. **일봉**과 **분봉(관측 간격이 일이 아닌 시계열)** 모두 업로드 가능하며, 행은 **타임스탬프 기준**으로 구분·정렬합니다(같은 날 여러 봉이 한 줄로 합쳐지지 않음).
- **실증 시나리오:** 키움 **영웅문글로벌** `가격데이터 시뮬레이션` 등에서 **엑셀저장 → CSV**로 보낸 **NQ000(미니 나스닥100 선물) 일간·분간** 형식을 1차 검증 대상으로 둡니다(`Skills.md` §1 참고 컬럼: `일자 / 시간`, `종가`, `거래량` 등).
- **지표 해석 전제:** 연율화 변동성의 \(\sqrt{252}\) 가정은 **일봉**에 가깝게 맞습니다. **분봉** 업로드 시 화면의 수익률·변동성은 “연속 봉 사이” 기준이 되므로, **일봉 리스크 지표**가 필요하면 소스 데이터를 일봉으로 맞출 것을 권장합니다.
- **미포함(확장 예정):** 다종목 포트폴리오 최적화, 실시간 시세 API, 매매 주문 연동, 시가·고가·저가·미결제약정 기반 전용 위젯.
- **법적·윤리 전제:** 투자 권유가 아닌 분석·시연 목적이며, 데이터 정확성은 사용자 책임으로 둡니다(`Skills.md` §7과 동일).

## 2. 분석 흐름 설계

전 과정은 **입력 → 검증·정규화 → 지표 → 시각화 결정 → 인사이트**의 파이프라인으로 고정합니다. 세부 수치·별칭은 `Skills.md`가 단일 출처(Single Source of Truth)입니다.

```
[CSV 업로드]
      ↓
[바이트 디코딩] → UTF-8 BOM 우선, 그 외 UTF-8 vs CP949(EUC-KR) 중 첫 줄 헤더 품질 점수로 자동 선택
      ↓
[헤더 파싱] ──→ BOM·공백 정규화(transformHeader) → Skills §1 별칭·공백 무시 매칭
              → 미매칭 시 키움형 8열(일시·시고저종·거래량·…) **위치 폴백**
      ↓
[전처리]   ──→ Skills §2 (날짜·시각 파싱, 동일 타임스탬프 병합, 숫자·과학적표기, 정렬)
      ↓
[지표 계산] ──→ Skills §3 (일간·누적 수익률, 연율화 변동성, MDD)
      ↓
[대시보드 조립] → Skills §4·§5 (위젯 종류·순서)
      ↓
[인사이트]  ──→ Skills §6 조건식 평가 → 해당 메시지 노출
```

### 2.1 단계별 산출물

| 단계 | 산출물 | 실패 시 |
|------|--------|---------|
| 매핑 | 사용자에게 표시할 **컬럼 매핑 결과**(어떤 헤더가 date/close인지) | 필수 필드 미매핑 시 오류 메시지, 차트 미생성 |
| 전처리 | 정렬된 시계열 행 배열 | **매핑은 성공했으나** 값 파싱 실패 시, **첫 행의 날짜·종가 샘플**을 안내 메시지에 포함해 원인 추적을 돕습니다. |
| 지표 | KPI용 스칼라 + 시계열용 배열 | 데이터 1행만 있을 경우 일부 지표는 0 또는 제한적 표시 |
| 인사이트 | 조건 만족 시 문구 리스트 | 없으면 “발화된 규칙 없음” 안내 |

## 3. 대시보드 구성

### 3.1 정보 구조 (IA)

화면은 **위에서 아래로 읽히는 리포트형** 한 페이지로 구성합니다. 심사·투표 환경에서 스크롤만으로 스토리가 완결되도록 했습니다.

1. **헤더 영역:** 서비스명, Skills·인코딩(UTF-8 권장 / CP949 자동) 안내, CSV 업로드·**기본 샘플**·**키움 NQ 형식 샘플** 버튼  
2. **매핑 카드:** Skills 별칭에 의해 선택된 실제 CSV 컬럼명 표시 → **범용성 증명**에 사용  
3. **KPI 카드 4종:** 누적 수익률, 연율화 변동성, MDD, 최근 종가  
4. **가격 라인 차트:** 종가 시계열  
5. **거래량 막대:** volume 매핑 성공 시에만 표시(선택 필드)  
6. **누적 수익률 영역 차트:** 구간 성과의 직관적 추이  
7. **최근 구간 수익률 막대:** 최근 **60스텝**(봉), 양·음 색 구분 — 일봉이면 약 60거래일, 분봉이면 동일 개수의 분 구간  
8. **인사이트 카드:** §6 규칙 충족 시 메시지·조건식 표시  
9. **푸터:** 비권유·교육용 문구

### 3.2 UI·UX 원칙

- **다크 톤 + 고대비 숫자:** 장시간 숫자 확인에 눈 부담 완화  
- **KPI는 큰 글자·부호 색:** 수익/손실 방향을 즉시 인지  
- **차트 축·툴팁에 단위 명시:** 오해 방지  
- **모바일:** 1차는 데스크톱 우선, 향후 그리드 열 수 조정으로 대응

### 3.3 Skills 문서와의 대응 표

| UI 블록 | Skills.md 근거 |
|---------|----------------|
| 컬럼 매핑 | §1 데이터 스키마 및 별칭 |
| 정렬·중복 처리 | §2 전처리 규칙 |
| KPI·시계열 값 | §3 지표 계산 규칙 |
| 차트 종류·순서 | §4 시각화 선택 기준, §5 대시보드 구성 규칙 |
| 경고 문구 | §6 인사이트 생성 규칙 |
| 면책 문구 | §7 해석 가드레일 |

## 4. Skills.md 설계 방향

### 4.1 문서의 역할

`Skills.md`는 **사람이 읽는 기준서**이자, 애플리케이션이 파싱하는 **구조화된 규격**의 역할을 동시에 수행합니다.

- **사람용:** 섹션·표·목록으로 분석 정책을 설명  
- **기계용:** `### 필드명` 블록의 별칭 목록, §6의 `조건 | 메시지` 한 줄 형식을 코드가 추출

### 4.2 설계 원칙

1. **단일 파일 우선:** 제출·리뷰가 단순하도록 v1.0은 파일 1개로 관리  
2. **확장 시 분할:** 역할별로 `01-schema.md`, `02-metrics.md` 등으로 쪼개 zip 제출 가능  
3. **조건식 안전성:** 인사이트 조건은 허용된 지표 키와 비교 연산자만 사용(임의 코드 실행 금지)  
4. **버전 표기:** 문서 상단에 버전·변경 시 영향 범위를 남길 것

### 4.3 바이브 코딩(문서 기반 개발) 연계

- 초기 UI·파이프라인·지표 함수의 **스캐폴딩**을 에이전트가 생성하고, 세부 수치·문구는 `Skills.md`와 동기화  
- 구현 시 “어느 섹션이 어느 모듈을 구동하는지”를 주석·README 수준으로 맞춰 **문서↔코드 추적 가능성** 확보

## 5. 확장 기능 아이디어

우선순위는 **데모 임팩트**와 **심사 항목(범용성·자동화)**에 가까운 순입니다.

| 우선순위 | 기능 | 기대 효과 |
|----------|------|-----------|
| — | **키움 NQ CSV 샘플·인코딩 자동 판별·8열 폴백** (v1.1 반영) | HTS에서 보낸 데이터와의 정합성·범용성 입증 |
| P1 | **여러 CSV 프리셋**(주식/펀드/암호자산 등) 원클릭 전환 | 범용성 시연 추가 강화 |
| P1 | **`Skills.md` 런타임 로드**(public 배치 후 fetch) | 문서만 교체해도 재빌드 없이 규칙 변경 시연 |
| P2 | **다종목 비교 모드**(동일 스키마 다파일) | 시각화 기준의 재사용 입증 |
| P2 | **리포트 PDF/PNG보내기** | 실무 활용 스토리 |
| P3 | **규칙 위반 로그**(어떤 행이 제외됐는지) | 데이터 품질·신뢰성 |
| P3 | **간단한 백테스트 모듈**(규칙 문서에 진입·청산 정의 추가) | Skills 확장성 과시 |

## 6. 기대 효과 및 마무리

본 서비스는 **“데이터 형식은 다양하지만, 해석 규격은 하나”**라는 메시지를 제품 형태로 보여 줍니다. 대시보드는 Skills 문서를 수정·버전 관리함으로써 **조직 표준 분석**으로 성장시킬 수 있으며, 해커톤 평가 관점에서도 범용성·Skills 설계·자동 생성·문서 기반 구현을 한 흐름으로 설명할 수 있습니다.
